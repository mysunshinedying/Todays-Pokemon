# Components 설정
- Loading / 로딩 화면
- Error / 에러 화면
- Inventory / 아이템 넣는 곳이고
- LuckyItem / 럭키아이템 정보
- MyBox / 포켓몬 저장
- PokemonDetail / 포켓몬 정보
- TypeBadge / 포켓몬 타입
- MenuBtn / 메뉴 버튼

## TodayPokemon / 오늘의 포켓몬
1. 오늘의 포켓몬 뽑기 창
   - 뽑기형 버튼

2. 오늘의 포켓몬 결과 창
결과창 내에 다음이 필요
   - 포켓몬 정보
   - 포획하기 → 포획 관련 컴포넌트 필요
   - 해당 포켓몬 관련 스토어 링크 → 컴포넌트 미필요

## MenuBtn / 메뉴 버튼
1. MyBox(박스를 보는 버튼)
2. 자기 인벤토리 보는 버튼(Inventory)
   - 이건 Menu Button으로 사용(컴포넌트 추가)
3. 설정
   - 세이브(백업) 로드(백업 불러오기)

## PokemonDetail / 포켓몬 정보
1. 이미지(API)
2. 타입(타입 뱃지 사용 → 컴포넌트)
3. 도감번호...?가 있으면 좋겠는데 전국번호로 나오게 해야겠고...
4. 키/몸무게
   - 이녀석은 MyBox에서도 사용됨.
5. 이름
6. 성별(암/수) → 랜덤
7. 성격 → 랜덤


## CatchPokemon / 포켓몬 포획
1. 몬스터볼(인벤토리에 있는 것)을 사용
2. 전부 사용하면 포획이 불가능해짐
3. 확률은 50% 정도? 면 어떨까..
4. 던지기!


## MyBox / 포켓몬 저장 박스
포켓몬 스프라이트
스프라이트를 누르면 다음 메뉴가 나타나야함
1. 정보보기 → 포켓몬 정보(컴포넌트)
2. 방생하기
3. 취소

MyBox의 고민되는 부분... → 특정 마리 수 이상이면 로컬스토리지에 저장하기 버겁지 않을까?하는 생각을..
박스는 1개, 30개로 두자.

## Inventory / 인벤토리
1. 럭키 아이템 소지창
2. 몬스터볼 소지창(첫 접속 시에는 몬스터볼 6개, 이후 접속 시마다 몇개를 추가지급하면 좋을지 고민되는데 2개가 적당할거같음.)

## LuckyItem / 럭키 아이템
Pokémon API에서 아이템 정보...를 불러오면 좋아보이긴하는데 고민
